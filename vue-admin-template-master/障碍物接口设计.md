# 障碍物坐标表接口设计

## 一、设计原则

障碍物表（Obstacle）设计为**ExperimentRecord的子资源**，与ExperimentStep保持一致的API设计模式：

1. **RESTful设计**：遵循统一的API设计规范
2. **子资源模式**：障碍物属于特定的实验记录
3. **批量操作**：支持批量添加和查询障碍物
4. **数据一致性**：障碍物与实验记录生命周期保持一致

## 二、数据库表结构

### Obstacle表（已添加到AI_guild.md）
```sql
CREATE TABLE `Obstacle` (
  `obstacle_id` INT PRIMARY KEY AUTO_INCREMENT,
  `record_id` INT NOT NULL,
  `x` INT NOT NULL COMMENT '障碍物X坐标',
  `y` INT NOT NULL COMMENT '障碍物Y坐标',
  `obstacle_type` ENUM('wall', 'water', 'mountain') DEFAULT 'wall' COMMENT '障碍物类型（可扩展）',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (`record_id`) REFERENCES `ExperimentRecord`(`record_id`)
    ON DELETE CASCADE ON UPDATE CASCADE,
  INDEX `idx_record_id` (`record_id`),
  INDEX `idx_coordinates` (`x`, `y`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实验障碍物坐标表';
```

## 三、后端接口设计

### 1. 障碍物管理接口（作为ExperimentRecord的子资源）

#### 1.1 获取实验记录的障碍物列表
- **方法**：GET
- **URL**：`/api/experiments/{recordId}/obstacles`
- **请求头**：`Authorization: Bearer {token}`
- **参数说明**：
  - `recordId`：实验记录ID（路径参数）
- **响应**：
```json
{
  "code": 200,
  "data": [
    {
      "obstacleId": 1,
      "recordId": 1,
      "x": 5,
      "y": 5,
      "obstacleType": "wall",
      "createdAt": "2026-01-29 10:00:00"
    },
    {
      "obstacleId": 2,
      "recordId": 1,
      "x": 5,
      "y": 6,
      "obstacleType": "wall",
      "createdAt": "2026-01-29 10:00:00"
    }
  ],
  "message": "成功"
}
```

#### 1.2 批量添加障碍物
- **方法**：POST
- **URL**：`/api/experiments/{recordId}/obstacles/batch`
- **请求头**：`Authorization: Bearer {token}`
- **请求体**：
```json
[
  {
    "x": 5,
    "y": 5,
    "obstacleType": "wall"
  },
  {
    "x": 5,
    "y": 6,
    "obstacleType": "wall"
  },
  {
    "x": 6,
    "y": 5,
    "obstacleType": "water"
  }
]
```
- **响应**：
```json
{
  "code": 200,
  "data": {
    "successCount": 3,
    "failedCount": 0,
    "totalCount": 3
  },
  "message": "成功添加3个障碍物"
}
```

#### 1.3 删除实验记录的所有障碍物
- **方法**：DELETE
- **URL**：`/api/experiments/{recordId}/obstacles`
- **请求头**：`Authorization: Bearer {token}`
- **响应**：
```json
{
  "code": 200,
  "data": {
    "deletedCount": 10
  },
  "message": "成功删除10个障碍物"
}
```

#### 1.4 删除单个障碍物
- **方法**：DELETE
- **URL**：`/api/experiments/{recordId}/obstacles/{obstacleId}`
- **请求头**：`Authorization: Bearer {token}`
- **响应**：
```json
{
  "code": 200,
  "data": null,
  "message": "成功删除障碍物"
}
```

## 五、Controller层设计

### ObstacleController
```java
package com.example.controller;

import com.example.dto.ObstacleDTO;
import com.example.service.ObstacleService;
import com.example.vo.ObstacleVO;
import com.example.common.Result;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import javax.validation.Valid;
import java.util.List;

@RestController
@RequestMapping("/api/experiments/{recordId}/obstacles")
public class ObstacleController {
    
    @Autowired
    private ObstacleService obstacleService;
    
    /**
     * 获取实验记录的障碍物列表
     */
    @GetMapping
    public Result<List<ObstacleVO>> getObstaclesByRecordId(@PathVariable Integer recordId) {
        List<ObstacleVO> obstacles = obstacleService.getObstaclesByRecordId(recordId);
        return Result.success(obstacles);
    }
    
    /**
     * 批量添加障碍物
     */
    @PostMapping("/batch")
    public Result<BatchResult> addObstaclesBatch(
            @PathVariable Integer recordId,
            @RequestBody @Valid List<ObstacleDTO> obstacleDTOs) {
        BatchResult result = obstacleService.addObstaclesBatch(recordId, obstacleDTOs);
        return Result.success(result);
    }
    
    /**
     * 删除实验记录的所有障碍物
     */
    @DeleteMapping
    public Result<DeleteResult> deleteObstaclesByRecordId(@PathVariable Integer recordId) {
        DeleteResult result = obstacleService.deleteObstaclesByRecordId(recordId);
        return Result.success(result);
    }
    
    /**
     * 删除单个障碍物
     */
    @DeleteMapping("/{obstacleId}")
    public Result<Void> deleteObstacle(
            @PathVariable Integer recordId,
            @PathVariable Integer obstacleId) {
        obstacleService.deleteObstacle(recordId, obstacleId);
        return Result.success();
    }
    
    // 批量操作结果类
    @Data
    public static class BatchResult {
        private int successCount;
        private int failedCount;
        private int totalCount;
    }
    
    // 删除结果类
    @Data
    public static class DeleteResult {
        private int deletedCount;
    }
}
```

## 九、数据迁移方案

### 1. 迁移现有数据
如果已有实验记录数据，需要从ExperimentStep表的gridState字段中提取障碍物信息：

```sql
-- 创建临时表存储从gridState提取的障碍物
CREATE TEMPORARY TABLE temp_obstacles AS
SELECT 
  es.record_id,
  JSON_EXTRACT(es.grid_state, '$.obstacles[*].x') as x,
  JSON_EXTRACT(es.grid_state, '$.obstacles[*].y') as y
FROM ExperimentStep es
WHERE es.grid_state IS NOT NULL
  AND JSON_EXTRACT(es.grid_state, '$.obstacles') IS NOT NULL;

-- 插入到Obstacle表
INSERT INTO Obstacle (record_id, x, y, obstacle_type)
SELECT record_id, x, y, 'wall'
FROM temp_obstacles;

-- 删除临时表
DROP TEMPORARY TABLE temp_obstacles;
```

## 十、对应关系总结

1. **Obstacle表** → `Obstacle`实体类 → `ObstacleController`

### 前端文件对应关系
