# 时间段搜索接口设计

## 需求背景
当前实验记录搜索功能中，开始时间和结束时间使用的是精确匹配（精确到秒），这在实际使用中不够灵活。用户希望能够搜索一个时间段内的实验记录，而不是精确匹配某个时间点。

## 当前API接口
当前实验记录列表接口：`GET /api/experiments`

当前支持的查询参数：
- `page`: 页码
- `size`: 每页大小
- `userId`: 用户ID
- `algorithmId`: 算法ID
- `status`: 实验状态
- `startTime`: 开始时间（精确匹配，格式：yyyy-MM-dd HH:mm:ss）
- `endTime`: 结束时间（精确匹配，格式：yyyy-MM-dd HH:mm:ss）

## 问题分析
当前接口的问题：
1. `startTime` 和 `endTime` 使用精确匹配，用户需要输入精确的时间
2. 无法搜索某个时间段内的记录（如：2026-01-01 到 2026-01-31）
3. 无法搜索开始时间在某个时间点之后的记录
4. 无法搜索结束时间在某个时间点之前的记录

## 解决方案
### 方案一：修改现有接口（推荐）
修改现有接口，支持时间段搜索：

#### 修改后的查询参数：
- `page`: 页码
- `size`: 每页大小
- `userId`: 用户ID
- `algorithmId`: 算法ID
- `status`: 实验状态
- `startTimeFrom`: 开始时间起始（可选）
- `startTimeTo`: 开始时间结束（可选）
- `endTimeFrom`: 结束时间起始（可选）
- `endTimeTo`: 结束时间结束（可选）

#### 查询逻辑：
1. 如果只提供 `startTimeFrom`：查询开始时间 >= `startTimeFrom` 的记录
2. 如果只提供 `startTimeTo`：查询开始时间 <= `startTimeTo` 的记录
3. 如果同时提供 `startTimeFrom` 和 `startTimeTo`：查询开始时间在 `[startTimeFrom, startTimeTo]` 范围内的记录
4. 结束时间同理

#### 向后兼容：
为了保持向后兼容，可以同时支持旧的 `startTime` 和 `endTime` 参数（精确匹配），但建议前端使用新的时间段参数。

### 方案二：新增接口
新增一个专门用于时间段搜索的接口：
- `GET /api/experiments/searchByTimeRange`

#### 查询参数：
- `page`: 页码
- `size`: 每页大小
- `userId`: 用户ID
- `algorithmId`: 算法ID
- `status`: 实验状态
- `startTimeFrom`: 开始时间起始
- `startTimeTo`: 开始时间结束
- `endTimeFrom`: 结束时间起始
- `endTimeTo`: 结束时间结束

## 前端修改建议
### 当前前端代码
当前前端代码中，开始时间和结束时间使用的是 `el-date-picker` 组件，返回的是精确的时间字符串。

### 修改建议
1. 保持现有的 `el-date-picker` 组件不变
2. 修改API调用参数：
   - 将 `startTime` 参数改为 `startTimeFrom`（如果用户选择了开始时间）
   - 将 `endTime` 参数改为 `endTimeTo`（如果用户选择了结束时间）
3. 或者同时支持两种方式，根据后端接口实现决定

## 后端实现建议
### Controller层修改
```java
@GetMapping("/experiments")
public Result getExperimentList(
    @RequestParam(required = false) Integer page,
    @RequestParam(required = false) Integer size,
    @RequestParam(required = false) Integer userId,
    @RequestParam(required = false) Integer algorithmId,
    @RequestParam(required = false) String status,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime startTime,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime endTime,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime startTimeFrom,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime startTimeTo,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime endTimeFrom,
    @RequestParam(required = false) @DateTimeFormat(pattern = "yyyy-MM-dd HH:mm:ss") LocalDateTime endTimeTo) {
    
    // 构建查询条件
    QueryWrapper<ExperimentRecord> queryWrapper = new QueryWrapper<>();
    
    // 原有条件
    if (userId != null) {
        queryWrapper.eq("user_id", userId);
    }
    if (algorithmId != null) {
        queryWrapper.eq("algorithm_id", algorithmId);
    }
    if (status != null) {
        queryWrapper.eq("status", status);
    }
    
    // 处理开始时间（向后兼容）
    if (startTime != null) {
        queryWrapper.eq("start_time", startTime);
    }
    
    // 处理开始时间段
    if (startTimeFrom != null) {
        queryWrapper.ge("start_time", startTimeFrom);
    }
    if (startTimeTo != null) {
        queryWrapper.le("start_time", startTimeTo);
    }
    
    // 处理结束时间（向后兼容）
    if (endTime != null) {
        queryWrapper.eq("end_time", endTime);
    }
    
    // 处理结束时间段
    if (endTimeFrom != null) {
        queryWrapper.ge("end_time", endTimeFrom);
    }
    if (endTimeTo != null) {
        queryWrapper.le("end_time", endTimeTo);
    }
    
    // 执行查询
    Page<ExperimentRecord> pageResult = experimentRecordService.page(
        new Page<>(page != null ? page : 1, size != null ? size : 10),
        queryWrapper
    );
    
    return Result.success(pageResult);
}
```

### Service层修改
Service层不需要修改，只需要在Controller层调整查询条件即可。

## 测试用例
### 测试场景1：搜索某个时间段内的实验记录
- 请求：`GET /api/experiments?startTimeFrom=2026-01-01 00:00:00&startTimeTo=2026-01-31 23:59:59`
- 预期：返回2026年1月份开始的实验记录

### 测试场景2：搜索开始时间在某个时间点之后的记录
- 请求：`GET /api/experiments?startTimeFrom=2026-01-15 00:00:00`
- 预期：返回2026年1月15日之后开始的实验记录

### 测试场景3：搜索结束时间在某个时间点之前的记录
- 请求：`GET /api/experiments?endTimeTo=2026-01-15 23:59:59`
- 预期：返回2026年1月15日之前结束的实验记录

## 注意事项
1. 保持向后兼容，避免影响现有功能
2. 时间格式统一使用 `yyyy-MM-dd HH:mm:ss`
3. 考虑时区问题，建议使用UTC时间或明确时区
4. 数据库查询性能优化，确保时间段查询有合适的索引
