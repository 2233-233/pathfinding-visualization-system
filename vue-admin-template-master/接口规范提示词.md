【Vue + axios + SpringBoot 对接防踩坑提示词】

我项目使用 Spring Boot + MyBatis-Plus + Vue + axios。

后端接口统一返回：

{ code, msg, data, success }


但前端 axios response interceptor 已经统一做了拆壳：

return res.data


因此所有 API 方法返回值 === 后端 R.data 字段内容。

你在生成任何前端代码时必须遵守：

不允许使用 response.code / response.msg / response.data

页面中一律写：

const result = await apiFn()
this.list = result || []


如果是分页接口，返回值本身就是：

{ records, total, current, size }


而不是 R<Page<T>>。

如果后端返回结构与前端消费方式不一致，
必须先指出“拦截器拆壳 or 页面逻辑假设错误”，
再给修改方案。

【前后端对接强约束提示词】

我项目中 axios 拦截器已经统一做了“拆壳”处理：

// response interceptor 中
if (res.data !== undefined && res.data !== null) {
return res.data
} else {
return res
}


这意味着：

所有 API 方法返回的 response 本身就是 data 字段的内容

前端页面中 不允许再使用 response.code / response.msg / response.data

你在生成任何 Vue 页面或 API 调用逻辑时，必须遵守：

把 response 直接当作数组或对象使用

严禁再写：

if (response.code === 200) { ... }
this.list = response.data


正确写法只能是：

const response = await apiFn()
this.list = response || []


另外：

后端统一返回 R<T> 结构（code/msg/data/success）

但前端一律只消费“被 axios 拦截器拆壳后的 data”

如果你发现返回结构和前端使用方式不匹配，
必须先指出“是拦截器拆壳问题 or 页面逻辑问题”，
再给修改方案，而不是硬凑字段名。

本项目采用 Spring Boot + MyBatis-Plus + Vue + axios。

后端接口统一返回结构：

{ code, msg, data, success }


但前端 axios response interceptor 已统一做“拆壳”处理：

return res.data


因此你在生成任何前端代码时，必须遵守以下规则：

1️⃣ 关于返回值使用（最重要）

所有 API 方法的返回值 === 后端 R.data 字段内容

页面中严禁出现：

response.code
response.msg
response.data


正确写法只能是：

const result = await apiFn()
this.list = result || []

2️⃣ 关于列表接口

不分页接口：
返回值本身就是数组：

[ { ... }, { ... } ]


分页接口：
返回值本身就是 Page 对象：

{ records, total, current, size }


页面中严禁再写：

response.data.records
response.data.total

3️⃣ 关于字段命名

前端字段名必须与后端实体 JSON 序列化结果一致
例如：

algorithmId

studentId

userId

不允许前端自行发明字段名

4️⃣ 关于查询参数

前端 params 的 key 必须与后端 @RequestParam 名完全一致

不允许“同时传 studentId 和 student_id 试试运气”

5️⃣ 关于空值兜底

所有列表赋值必须兜底：

this.list = result || []


如果你发现返回结构与页面逻辑不匹配：

必须先判断是：

axios 拦截器拆壳问题

还是页面逻辑假设错误

再给修改方案，而不是硬凑字段名。
