3.3数据库设计
3.3.1数据库ER设计
本系统采用MySQL数据库进行数据存储与管理，根据系统功能需求设计了以下数据表：用户表（user）、题目表（problem）、题目完成记录表（problemcompletion）、实验记录表（experimentrecord）和实验步骤表（experimentstep）。
数据库视图关系模式：
用户：编号、账号、密码、角色、邮箱、姓名、学号、注册时间。
题目：编号、标题、描述、难度、算法编号、力扣链接。
题目完成记录：编号、用户编号、题目编号、完成状态、尝试次数、是否解决、笔记、最近复习日期、难度评分、花费时间（分钟）、创建时间、更新时间。
实验记录：编号、用户编号、题目编号、算法编号、开始时间、结束时间、路径长度、访问节点数、状态。
实验步骤：编号、记录编号、步骤序号、节点横坐标、节点纵坐标、节点状态、网格状态。
3.3.2数据库表结构设计
1）用户表（user）
用户表用于存储系统所有用户的基本信息，包括学生用户和管理员用户。
表名：user
字段名	类型	约束	说明
编号	int	主键、自增	用户ID
账号	varchar(50)	非空	用户名
密码	varchar(100)	非空	密码
角色	enum('student','admin')	非空	用户角色
邮箱	varchar(100)	可空	邮箱
姓名	varchar(100)	可空	用户姓名
学号	varchar(50)	可空	学号
注册时间	datetime	默认当前时间	注册时间
2）题目表（problem）
题目表用于存储题库中的算法练习题信息。
表名：problem
字段名	类型	约束	说明
编号	int	主键、自增	题目ID
标题	varchar(100)	非空	题目标题
描述	text	可空	题目描述
难度	enum('easy','medium','hard')	非空	难度级别
算法编号	int	非空	关联算法ID
力扣链接	varchar(255)	非空	力扣题目链接
3）题目完成记录表（problemcompletion）
题目完成记录表用于记录学生对每道题目的学习进度和完成情况。
表名：problemcompletion
字段名	类型	约束	说明
编号	int	主键、自增	记录ID
用户编号	int	非空	用户ID
题目编号	int	非空	题目ID
完成状态	enum('not_started','in_progress','completed')	默认'not_started'	完成状态
尝试次数	int	默认0	尝试次数
是否解决	tinyint(1)	默认0	是否已解决
笔记	text	可空	笔记
最近复习日期	date	可空	最近复习日期
难度评分	tinyint	可空	难度自评
花费时间（分钟）	int	默认0	花费时间
创建时间	datetime	默认当前时间	创建时间
更新时间	datetime	更新时自动更新	更新时间
4）实验记录表（experimentrecord）
实验记录表用于存储用户每次运行算法实验的完整记录。
表名：experimentrecord
字段名	类型	约束	说明
编号	int	主键、自增	记录ID
用户编号	int	非空	用户ID
题目编号	int	非空	题目ID
算法编号	int	非空	算法ID
开始时间	datetime	可空	开始时间
结束时间	datetime	可空	结束时间
路径长度	int	可空	路径长度
访问节点数	int	可空	访问节点数
状态	enum('completed','failed')	非空	实验状态
5）实验步骤表（experimentstep）
实验步骤表用于存储算法运行过程中每一步的详细状态数据，支持实验回放功能。
表名：experimentstep
字段名	类型	约束	说明
编号	int	主键、自增	步骤ID
记录编号	int	非空	实验记录ID
步骤序号	int	非空	步骤序号
节点横坐标	int	可空	节点X坐标
节点纵坐标	int	可空	节点Y坐标
节点状态	enum('visited','path','unvisited')	可空	节点状态
网格状态	json	可空	网格状态
3.3.3数据库ER图绘制提示词
创建一个数据库ER图（实体-关系图），白底黑字，标题为"算法可视化学习平台数据库ER图"。采用标准ER图风格，矩形框表示实体，菱形框表示关系，椭圆框表示属性，连线表示实体间的关系，连线上标注关系类型（1:1、1:N、M:N）。
实体列表：
1. 用户（user）：属性有编号（主键）、账号、密码、角色、邮箱、姓名、学号、注册时间。
2. 题目（problem）：属性有编号（主键）、标题、描述、难度、算法编号（外键）、力扣链接。
3. 题目完成记录（problemcompletion）：属性有编号（主键）、用户编号（外键）、题目编号（外键）、完成状态、尝试次数、是否解决、笔记、最近复习日期、难度评分、花费时间（分钟）、创建时间、更新时间。
4. 实验记录（experimentrecord）：属性有编号（主键）、用户编号（外键）、题目编号（外键）、算法编号（外键）、开始时间、结束时间、路径长度、访问节点数、状态。
5. 实验步骤（experimentstep）：属性有编号（主键）、记录编号（外键）、步骤序号、节点横坐标、节点纵坐标、节点状态、网格状态。
实体间关系：
1. 用户（user）与题目完成记录（problemcompletion）：一对多（1:N）关系。一个用户可以拥有多条题目完成记录，每条题目完成记录属于一个用户。
2. 用户（user）与实验记录（experimentrecord）：一对多（1:N）关系。一个用户可以拥有多条实验记录，每条实验记录属于一个用户。
3. 题目（problem）与题目完成记录（problemcompletion）：一对多（1:N）关系。一个题目可以对应多条完成记录，每条完成记录对应一个题目。
4. 题目（problem）与实验记录（experimentrecord）：一对多（1:N）关系。一个题目可以对应多条实验记录，每条实验记录对应一个题目。
5. 实验记录（experimentrecord）与实验步骤（experimentstep）：一对多（1:N）关系。一条实验记录可以包含多个实验步骤，每个实验步骤属于一条实验记录。
